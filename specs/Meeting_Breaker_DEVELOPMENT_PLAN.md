# Meeting Breaker — план разработки для агента

## 1. Назначение

Этот файл определяет порядок разработки MVP.

Агент должен выполнять этапы последовательно. После каждого этапа проект должен запускаться, собираться и не содержать TypeScript-ошибок.

Перед началом агент обязан прочитать `Meeting_Breaker_SPEC.md`.

## 2. Общие правила

- Не выходить за рамки MVP.
- Не добавлять backend.
- Не подключать Outlook API.
- Не реализовывать авторизацию.
- Использовать строгий TypeScript.
- Отделять игровую конфигурацию от логики.
- Не хардкодить встречи в сцене.
- Не переписывать рабочую архитектуру без причины.
- После каждого этапа выполнять доступные проверки.
- Не переходить дальше, пока текущий этап не работает.

- 09:00;
- 17:30;
- высота 15 минут;
- высота 120 минут.

### Результат

## 11. Этап 8. Кофе

### Задачи

- начальное число чашек;
- расход чашки при падении шарика;
- `COFFEE_CONSUMED`;
- `COFFEE_CHANGED`;
- пауза после потери;
- возврат шарика;
- поражение при нуле.

### Алгоритм

1. Шарик падает.
2. Игра останавливается.
3. Расходуется чашка.
4. Показывается случайная фраза.
5. Если кофе осталось — новый шарик.
6. Если нет — `GAME_OVER`.

### Тесты

- уменьшение кофе;
- отсутствие отрицательных значений;
- повторный запуск;
- поражение после последней чашки.

### Результат

В коде нет `lives`, используется `coffeeCups`.

## 15. Этап 12. Бонусы

Создать:

- `PowerUp`;
- `PowerUpSystem`;
- выпадение;
- падение;
- подбор;
- событие активации.

### Async Mode

- дополнительный шарик;
- поражение только после потери всех шариков;
- очистка дополнительных шариков при перезапуске.

### Decline

- следующее попадание уничтожает любую встречу;
- эффект расходуется один раз;
- есть индикатор в HUD.

### Espresso Shot

- увеличить платформу;
- уменьшить скорость шарика;
- длительность 10 секунд;
- плавный возврат;
- повторная активация обновляет таймер;
- эффект не складывается.

### Тесты

- активация;
- завершение таймера;
- повторная активация;
- отсутствие наложения;
- расход Decline.
