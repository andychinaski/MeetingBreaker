# Meeting Breaker — UX, управление и обучение: спецификация доработки

## 1. Цель

Доработать пользовательский сценарий игры и устранить текущие UX-проблемы:

- убрать выбор уровня с главного экрана;
- корректно разделить прохождение и endless-режимы;
- починить выбор схемы управления;
- убрать хардкод пользовательских текстов;
- упростить внешний вид шара и платформы;
- вынести HUD за пределы игрового поля;
- полностью переделать обучающий уровень;
- увеличить читаемость названий встреч.

Эта работа является отдельным этапом после текущих hotfix и не должна смешиваться с добавлением новых специальных типов встреч.

---

## 2. Ограничения

В рамках этой доработки не менять:

- баланс уровней;
- набор MVP-бонусов;
- правила начисления очков;
- серверную часть;
- localStorage leaderboard;
- post-MVP механики встреч;
- физику столкновений, кроме подключения выбранной схемы управления;
- основную систему тем, кроме адаптации новых UI-компонентов.

# Часть I. Главное меню и выбор режима

## 3. Убрать выбор уровня с главного экрана

Текущий выбор уровня на главном экране удалить.

Причина:

- уровень имеет смысл только для режима `Прохождение`;
- Relax и Hard являются endless-режимами;
- пользователь не должен выбирать уровень до выбора режима.

### Новый flow

```text
Главное меню
└── Играть
    ├── Прохождение
    │   └── Выбор уровня / продолжение кампании
    ├── Relax Mode
    │   └── Запуск endless-сессии
    └── Hard Mode
        └── Запуск endless-сессии
```

### Требования

- на главном экране нет select уровня;
- после нажатия `Играть` открывается выбор режима;
- при выборе `Прохождение` открывается выбор уровня;
- при выборе Relax или Hard сессия запускается без выбора уровня;
- выбранный режим должен быть явно виден перед запуском.

## 4. Информация на главном экране

Допускается оставить:

- приветствие;
- общий лучший результат;
- максимум освобождённого времени;
- кнопку `Играть`;
- таблицу лидеров;
- настройки;
- информацию.

Если нужен дополнительный блок, показывать:

```text
Последняя игра
Прохождение · Неделя 3
```

Но не возвращать select уровня на главный экран.

# Часть II. Схема управления

## 5. Обязательный выбор управления

Настройка управления должна реально влиять на игру.

Поддерживаемые схемы:

```ts
type ControlScheme = 'keyboard' | 'mouse';
```

### Клавиатура

Разрешено:

- `A` / `D`;
- `←` / `→`;
- `Space` для запуска шара.

Запрещено:

- движение платформы мышью;
- запуск шара кликом.

### Мышь

Разрешено:

- движение платформы за курсором;
- запуск шара кликом.

Запрещено:

- движение платформы клавишами;
- запуск шара через `Space`.

Мышь при этом продолжает работать в интерфейсе меню и модальных окон.

## 6. Первый запуск

Перед запуском первой игровой сессии проверить настройку:

```ts
controlScheme
```

Если она отсутствует:

1. Показать обязательное окно выбора.
2. Предложить:
   - `Клавиатура`;
   - `Мышь`.
3. Сохранить выбор.
4. Только после этого продолжить запуск игры.

Не назначать схему молча по умолчанию.

## 7. Изменение схемы в настройках

При изменении схемы:

- отключить обработчики старой схемы;
- подключить обработчики новой схемы;
- не допускать одновременной работы мыши и клавиатуры;
- настройка должна применяться к следующей сессии;
- если архитектура позволяет — применять сразу в активной игре после закрытия настроек.

## 8. InputController

Вынести управление в отдельный контроллер.

```ts
export interface InputController {
  enable(): void;
  disable(): void;
  destroy(): void;
}
```

Реализации:

```ts
KeyboardInputController
MouseInputController
```

`GameScene` не должна одновременно подписываться на обе схемы.

- конфигурациях уровней;

- HUD;

# Часть IV. Шар и платформа

## 13. Убрать текст с шара

Удалить надпись:

```text
Task
```

Шар должен быть:

- однотонным;
- хорошо заметным на обеих темах;
- без текста;
- без лишней графики.

Допускаются:

- лёгкий градиент;
- тень;
- свечение;
- контур.

## 14. Убрать текст с платформы

Удалить надпись:

```text
FOCUS TIME
```

Платформа должна быть:

- простой;
- контрастной;
- без текста;
- хорошо видимой в светлой и тёмной теме.

# Часть V. HUD над игровым полем

## 15. Новая компоновка

HUD вынести из Phaser canvas и разместить над игровым полем.

```text
[ Уровень / Волна ] [ Очки ] [ Свободно ] [ Комбо ] [ Кофе ] [ Пауза ]

┌────────────────────────────────────────────┐
│                                            │
│               Игровое поле                 │
│                                            │
└────────────────────────────────────────────┘
```

## 16. Контекст HUD по режимам

### Прохождение

Показывать:

- уровень;
- очки;
- свободное время;
- комбо;
- кофе;
- паузу.

### Relax Mode

Показывать:

- волну;
- очки;
- свободное время;
- комбо;
- кофе;
- паузу.

### Hard Mode

Показывать:

- волну;
- очки;
- свободное время;
- комбо;
- паузу.

Не показывать кофе в Hard Mode.

## 17. Архитектура

- HUD реализуется на React;
- Phaser передаёт данные через event bus;
- HUD не находится поверх игрового поля;
- игровая область получает всю доступную высоту canvas;
- HUD не должен вызывать React render на каждом кадре;
- обновлять только изменённые значения.

# Часть VI. Размер текста в блоках встреч

## 18. Увеличить размер названий

Увеличить шрифт названия встречи на `2px` относительно текущего значения.

Пример:

```ts
meetingTitleFontSize = currentMeetingTitleFontSize + 2;
```

Требования:

- применить ко всем блокам встреч;
- сохранить кастомный акцентный шрифт, если он используется;
- не увеличивать автоматически остальные подписи;
- проверить длинные названия;
- проверить блоки высотой 15 и 30 минут;
- текст не должен выходить за границы блока.

## 19. Правила отображения

### Крупные блоки

Показывать:

- название;
- время;
- участников, если помещаются.

### Средние блоки

Показывать:

- название;
- время.

### Маленькие блоки

Показывать:

- название;
- допускается сокращение;
- скрыть вторичные данные.

Если название не помещается:

- использовать ellipsis;
- не уменьшать шрифт ниже базового;
- не выводить текст за границы.

Для Phaser использовать обрезку текста или сокращённое отображаемое имя.

# Часть VII. Полная переделка обучения

## 20. Цель

Текущий обучающий уровень заменить.

Обучение должно:

- вести пользователя по шагам;
- останавливать обычный игровой процесс;
- фокусировать внимание на одном элементе;
- не перегружать UI;
- ждать конкретного действия пользователя;
- использовать выбранную схему управления.

## 21. Основной UX шага

Каждый шаг состоит из двух фаз.

### Фаза объяснения

1. Игра поставлена на паузу.
2. Поле затемнено.
3. Нужный объект выделен spotlight.
4. По центру показывается крупный текст.
5. Пользователь подтверждает продолжение.

### Фаза действия

1. Текст уменьшается или перемещается.
2. Подсвеченный объект остаётся видимым.
3. Разрешается только нужное действие.
4. Остальные действия блокируются.
5. Контроллер ждёт нужное игровое событие.
6. После события запускается следующий шаг.

Не оставлять большой текст по центру во время активного движения шара, если он закрывает игровое поле.

## 22. Overlay и spotlight

Создать React overlay поверх игрового canvas.

Требования:

- затемнение всей сцены;
- прозрачный вырез вокруг целевого элемента;
- плавное появление;
- возможность подсветить:
  - платформу;
  - шар;
  - встречу;
  - бонус;
  - HUD;
  - кнопку паузы;
- overlay не должен ломать resize;
- координаты Phaser-объекта преобразуются в координаты экрана.

## 23. Сценарий обучения

### Шаг 1. Платформа

Текст:

```text
Это твоя платформа.
```

Затем инструкция зависит от схемы:

```text
Передвинь её мышью.
```

или:

```text
Передвинь её клавишами.
```

Событие:

```text
PADDLE_MOVED
```

### Шаг 2. Запуск шара

Подсветить шар.

Для мыши:

```text
Кликни, чтобы запустить шар.
```

Для клавиатуры:

```text
Нажми Space, чтобы запустить шар.
```

Событие:

```text
BALL_LAUNCHED
```

### Шаг 3. Обычная встреча

Оставить одну встречу с 1 HP.

```text
Разбей встречу.
```

Событие:

```text
MEETING_DESTROYED
```

### Шаг 4. Прочная встреча

Создать встречу с 2 HP.

```text
Некоторые встречи требуют нескольких попаданий.
```

После первого попадания:

- сделать короткую паузу;
- показать повреждённое состояние;
- продолжить.

### Шаг 5. HUD

Поочерёдно подсветить:

- очки;
- свободное время;
- комбо.

Не объяснять все элементы одним большим абзацем.

### Шаг 6. Кофе

Не ждать случайной ошибки.

Сценарно вызвать потерю шара или управляемую демонстрацию.

```text
Потерял шар — потратил чашку кофе.
```

Подсветить счётчик кофе.

### Шаг 7. Бонус

Создать гарантированный Espresso Shot.

```text
Поймай бонус.
```

Событие:

```text
POWER_UP_COLLECTED
```

### Шаг 8. Пауза

Подсветить кнопку паузы.

```text
Игру можно поставить на паузу.
```

Событие:

```text
GAME_PAUSED
```

### Шаг 9. Самостоятельная часть

Создать 3–5 простых статических встреч.

```text
Теперь очисти календарь самостоятельно.
```

После уничтожения всех:

```text
Обучение завершено.
```

## 24. TutorialController

```ts
export interface TutorialStep {
  id: string;
  messageKey: TranslationKey;
  actionMessageKey?: TranslationKey;

  spotlightTarget?: TutorialTarget;
  allowedActions: TutorialAction[];
  completionEvent?: GameEvent;

  setup?: TutorialSetup;
  cleanup?: TutorialCleanup;

```

Контроллер должен:

- запускать шаги по порядку;
- подписываться только на нужное событие;
- не переходить на следующий шаг дважды;
- блокировать лишние действия;
- очищать обработчики;
- поддерживать skip;
- учитывать выбранную схему управления;
- корректно работать после resize.

## 25. Управление обучением

Добавить:

- кнопку `Пропустить обучение`;
- подтверждение пропуска;
- повторный запуск из настроек;
- сохранение:

```ts

```

## 35. Unit-тесты

Покрыть:

- выбор input controller;
- отключение старых обработчиков;
- отсутствие control scheme;
- перевод ключей;
- fallback локализации;

- однократное выполнение событий;

## 36. Интеграционные тесты

Проверить:

- мышь не управляет игрой в keyboard mode;
- клавиатура не управляет платформой в mouse mode;
- настройка сохраняется;
- при отсутствии настройки появляется modal;
- Campaign показывает уровень;
- Relax и Hard не показывают уровень;
- HUD не перекрывает canvas;
- смена языка обновляет Phaser и React;
- tutorial блокирует лишние действия;
- tutorial использует корректные инструкции управления.
