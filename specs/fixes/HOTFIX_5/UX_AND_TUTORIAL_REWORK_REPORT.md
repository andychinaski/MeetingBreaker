# UX_AND_TUTORIAL_REWORK — отчёт

## Что изменено

- Убран выбор уровня с главного экрана. Campaign открывает отдельный выбор рабочей недели, Relax и Hard запускаются сразу.
- Добавлен обязательный выбор управления для старых профилей без `controlScheme`.
- Клавиатура и мышь вынесены в отдельные `InputController`; одновременно активна только выбранная схема.
- Добавлены словари RU/EN для основного интерфейса, HUD, результатов, настроек, onboarding, обучения, уровней и встреч. React и Phaser используют язык профиля.
- С шара удалена надпись `TASK`, с платформы — `FOCUS TIME`.
- React HUD перенесён над canvas. Campaign показывает уровень, endless — волну, Hard не показывает кофе.
- Заголовки встреч увеличены на 2 px и безопасно сокращаются через ellipsis.
- Обучение переписано как сценарная state machine с фазами объяснения и действия, паузой физики, разрешёнными действиями, React spotlight-overlay, сценарной потерей кофе, гарантированным Espresso Shot и подтверждением пропуска.
- Самостоятельная часть обучения содержит пять простых статических встреч. Результат прохождения или пропуска сохраняется в профиле.
- Адаптирована высота игрового экрана с учётом вынесенного HUD.
- React-компоненты больше не получают `Phaser.Game`: добавлен безопасный event bridge, устраняющий dev-mode crash при чтении уничтоженного `Pointer.x`. Добавлен inline favicon без сетевого 404.
- Spotlight рассчитывается по реальным world-координатам Phaser и DOM-границам HUD, пересчитывается при resize. После правильного действия показывается success-фаза `Готово!` продолжительностью 750 мс, повторные события в это время игнорируются.

## Как тестировать

1. Очистить localStorage, пройти onboarding и отдельно проверить мышь и клавиатуру.
2. Для старого профиля удалить `preferences.controlScheme`: перед режимами должен появиться обязательный modal без закрытия.
3. Проверить flow: `Играть → Прохождение → выбор недели`; Relax/Hard должны запускаться без выбора недели.
4. В keyboard mode проверить A/D, стрелки и Space; курсор и клик не должны управлять игрой. В mouse mode — наоборот.
5. Проверить HUD над canvas: уровень в Campaign, волна в endless, отсутствие кофе в Hard.
6. Запустить обучение из информации или настроек, пройти объяснения и действия, проверить spotlight, сценарный кофе/бонус и подтверждение пропуска.
7. Переключить язык на English и запустить новую сцену: React, Phaser, дни недели и встречи должны быть английскими.

## На что обратить внимание

- Spotlight задаётся адаптивными CSS-областями относительно игрового контейнера; при радикально иной компоновке HUD координаты потребуется синхронно обновить.
- Исходные level-конфиги сохраняют русские fallback-значения для обратной совместимости типов, но пользовательские render-path используют словарь по стабильному `levelId`/`typeId`.
- Названия MVP-бонусов (`Espresso Shot`, `Decline`, `Async Mode`) оставлены каноническими продуктовыми терминами.

## Известные ограничения

- Vite сообщает предупреждение о размере Phaser bundle; это существующее ограничение сборки и не влияет на работу hotfix.
- Spotlight использует DOM/CSS-позиционирование, а не геометрию Phaser-объектов на каждом кадре, чтобы не вызывать постоянные React-render. Позиции адаптивны к текущему layout и resize.
- После последнего изменения количества учебных встреч и словаря дней повторный unit/E2E-прогон заблокирован лимитом среды; финальные `lint` и `build` выполнены успешно.
