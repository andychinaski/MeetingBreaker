# HOTFIX 3

## Цель

Исправить цветовую схему, компоновку главного меню, поведение разделов меню и применение светлой темы внутри игровой сцены.

Референсы:

- `image1.png` — текущее состояние;
- `image2.png` — целевая компоновка.

## 1. Новый акцентный цвет

Заменить текущий зелёный акцент на голубой в стиле Outlook / Microsoft Fluent.

Рекомендуемые токены:

```css
--accent-primary: #0f6cbd;
--accent-hover: #115ea3;
--accent-active: #0c3b5e;
--accent-text: #ffffff;
--accent-focus: #2886de;
```

Требования:

- не хардкодить цвета в компонентах;
- обновить кнопки, ссылки, выделения, focus-state и акцентные подписи;
- проверить светлую и тёмную тему;
- текст на акцентных кнопках должен быть белым и контрастным;
- состояния `default`, `hover`, `active`, `focus`, `disabled` должны различаться.

## 2. Переделать главное меню по `image2.png`

Текущее состояние показано на `image1.png`, целевое — на `image2.png`.

Требуемая компоновка:

1. Приветствие пользователя.
2. Текст: `Разбей все встречи и сохрани кофе до пятницы.`
3. Две карточки статистики в одну строку:
   - лучший результат;
   - максимум свободного времени.
4. Поле выбора уровня на всю ширину.
5. Вертикальное меню:
   - `Играть`;
   - `Таблица лидеров`;
   - `Настройки`;
   - `Информация`.

Изменения:

- убрать двухколоночную раскладку основной карточки;
- убрать крупный внутренний заголовок `Освободи время для настоящей работы`;
- сделать карточку компактной, вертикальной и центрированной;
- кнопки занимают всю ширину;
- `Играть` остаётся акцентной;
- остальные кнопки одинакового размера;
- сохранить нормальные вертикальные отступы.

Адаптивность:

- на широком экране карточка остаётся компактной;
- на узком занимает доступную ширину с боковыми отступами;
- статистика переходит в одну колонку только при нехватке места.

## 3. Переделать меню выбора режима

Оформить его в той же системе, что и главное меню:

- компактная центральная карточка;
- режимы расположены вертикально;
- каждый режим занимает всю ширину;
- название и краткое описание находятся внутри одного элемента;
- выбранный режим имеет заметное акцентное состояние;
- кнопка возврата возвращает в главное меню без перезагрузки.

Режимы:

1. `Прохождение`
2. `Relax Mode`
3. `Hard Mode`

Не использовать широкую двухколоночную раскладку.

## 4. Таблица лидеров и информация — модальные окна

Пункты `Таблица лидеров` и `Информация` должны открываться так же, как `Настройки`: поверх текущего меню, без перехода на отдельный route.

Требования к общему modal-компоненту:

- затемнение фона;
- кнопка закрытия;
- закрытие по `Esc`;
- закрытие по backdrop, если так уже работает у настроек;
- focus trap;
- возврат фокуса на кнопку, открывшую окно;
- отсутствие перезагрузки;
- один переиспользуемый modal-компонент для настроек, лидерборда и информации.

## 5. Исправить светлую тему внутри игры

Сейчас светлая тема не применяется к:

- игровому полю;
- popup внутри игровой сцены.

В светлой теме должны меняться:

- фон Phaser canvas / сцены;
- календарная сетка;
- линии;
- подписи дней и времени;
- границы игрового поля;
- служебные панели;
- popup паузы;
- popup победы и поражения;
- popup потери мяча / расхода кофе;
- tutorial popup;
- внутриигровые подсказки.

Popup в светлой теме:

- светлый фон;
- тёмный текст;
- заметная граница или тень;
- голубой акцент;
- читаемые кнопки.

Архитектура:

- React и Phaser используют единый источник темы;
- активная Phaser-сцена получает событие изменения темы;
- сцена обновляется без обязательного restart;
- новые объекты создаются с текущей темой;
- не хранить отдельную несинхронизированную тему внутри Phaser.

## 6. Что не менять

Не менять:

- физику;
- генерацию уровней;
- типы встреч;
- баланс;
- бонусы;
- правила режимов;
- onboarding;
- структуру localStorage, кроме необходимости для темы.

## 7. Ручное тестирование

### Главное меню

1. Сравнить результат с `image2.png`.
2. Проверить компактную центральную карточку.
3. Проверить вертикальное меню.
4. Проверить отсутствие большого внутреннего заголовка.
5. Проверить обе темы.

### Выбор режима

1. Нажать `Играть`.
2. Проверить вертикальный список режимов.
3. Проверить описания.
4. Проверить выбор и возврат.

### Модальные окна

1. Открыть настройки.
2. Открыть таблицу лидеров.
3. Открыть информацию.
4. Проверить одинаковое поведение.
5. Проверить `Esc`, backdrop и возврат фокуса.

### Темы

1. Запустить игру в тёмной теме.
2. Переключить на светлую без restart.
3. Проверить фон, сетку и подписи.
4. Проверить паузу, победу, поражение и tutorial popup.
5. Переключить тему обратно.

### Цвета

1. Проверить новый голубой акцент.
2. Проверить текст акцентных кнопок.
3. Проверить `hover`, `active`, `focus`, `disabled`.

## 8. Автоматические проверки

```bash
npm run lint
npm run build
```

```bash

```

- открытия и закрытия модальных окон;
- возврата фокуса;
- переключения темы;
- передачи темы в Phaser;
- вертикальной структуры меню;
- открытия выбора режима.
