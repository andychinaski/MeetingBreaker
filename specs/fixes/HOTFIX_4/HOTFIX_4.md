# HOTFIX 4

## Цель

Исправить выравнивание header, отступы и типографику в главном меню, настройках и меню выбора игрового режима.

Референс:

- `reference.png` — текущее состояние и ориентир по компоновке.

---

## 1. Отцентрировать header игры

Отцентрировать весь верхний header относительно основной контентной колонки и страницы.

В header входят:

- `ДАЙ КАЛЕНДАРЮ ПИЗДЫ`;
- `Meeting Breaker`;
- подзаголовок.

Требования:

- использовать общий центрированный контейнер;
- применить `text-align: center`;
- header и основная карточка должны находиться на одной центральной оси;
- не выравнивать header по левому краю страницы;
- сохранить адаптивность;
- на узких экранах текст не должен выходить за границы.

---

## 2. Добавить отступ перед кнопкой «Повторить обучение»

В модальном окне `Настройки` кнопка:

```text
Повторить обучение
```

прижата к предыдущему элементу.

Нужно:

- добавить заметный `margin-top`;
- использовать существующий spacing token, если он есть;
- не хардкодить случайное значение только для одного разрешения;
- проверить светлую и тёмную тему.

Рекомендуемый визуальный отступ: 16–24 px.

---

## 3. Увеличить размер акцентных подписей

Увеличить размер шрифта примерно в 2 раза для:

```text
Привет, {username}
```

и:

```text
Игровой режим
```

Требования:

- использовать общий CSS-класс акцентной подписи;
- допустимо, если изменение этого общего класса затронет другие аналогичные акцентные подписи;
- сохранить uppercase и текущий letter-spacing;
- проверить, что текст не ломает карточку на узких экранах;
- не увеличивать описание и обычный текст вместе с подписью.

---

## 4. Привести шрифты к единой системе

Сейчас кастомный и стандартный шрифты используются непоследовательно.

Правило:

### Кастомный шрифт

Использовать существующий кастомный шрифт для акцентных элементов:

- верхний kicker;
- акцентные uppercase-подписи;
- названия секций;
- названия режимов;
- заголовки карточек статистики;
- другие визуально акцентные элементы.

### Стандартный шрифт

Использовать стандартный основной шрифт для:

- описаний;
- обычных абзацев;
- текста select;
- вспомогательных подписей;
- пояснений;
- неакцентного интерфейсного текста.

Требования:

- не подключать новый шрифт;
- использовать уже существующий custom font;
- определить или переиспользовать typography tokens/classes;
- не смешивать два шрифта внутри одной текстовой роли;
- проверить главное меню, настройки, информацию, лидерборд и выбор режима.

---

## 5. Исправить типографику выбора игрового режима

В карточках выбора режима:

### Название режима

- увеличить текущий размер шрифта на `2px`;
- оставить акцентным;
- использовать кастомный шрифт;
- сохранить заметный вес.

### Описание режима

- использовать стандартный шрифт;
- установить `font-weight: 400`;
- не использовать bold или semi-bold;
- сохранить нормальную читаемость и межстрочный интервал.

Пример:

```css
.mode-title {
  font-size: calc(var(--current-mode-title-size) + 2px);
  font-family: var(--font-accent);
  font-weight: 700;
}

.mode-description {
  font-family: var(--font-body);
  font-weight: 400;
}
```

Не обязательно использовать именно эти имена переменных, если в проекте уже есть своя система.

---

## 6. Что не менять

В рамках HOTFIX 4 не менять:

- игровую логику;
- физику;
- генерацию уровней;
- режимы;
- баланс;
- темы и цвета, кроме необходимых CSS-правок типографики;
- структуру модальных окон;
- localStorage.

---

## 7. Ручное тестирование

### Header

1. Открыть главное меню.
2. Проверить, что kicker, title и subtitle центрированы.
3. Проверить совпадение центральной оси header и основной карточки.
4. Проверить широкий и узкий экран.

### Настройки

1. Открыть `Настройки`.
2. Найти кнопку `Повторить обучение`.
3. Проверить отступ сверху.
4. Проверить обе темы.

### Типографика

1. Проверить `Привет, {username}`.
2. Проверить `Игровой режим`.
3. Проверить акцентные подписи.
4. Проверить обычные описания.
5. Убедиться, что кастомный шрифт используется только в акцентных ролях.

### Выбор режима

1. Открыть меню выбора режима.
2. Проверить увеличение названий на 2 px.
3. Проверить кастомный шрифт заголовков.
4. Проверить обычный шрифт описаний.
5. Убедиться, что описание имеет `font-weight: 400`.
