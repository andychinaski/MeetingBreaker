# Meeting Breaker — общая спецификация проекта

## 1. Концепция

**Meeting Breaker** — браузерная игра в жанре Arkanoid, где игрок разбивает шариком блоки, стилизованные под встречи в календаре IT-компании.

Календарь используется только как визуальная метафора. Интеграции с Outlook, Google Calendar и другими сервисами нет.

Основные метафоры:

- шарик — рабочая задача;
- платформа — фокус разработчика;
- встречи — препятствия;
- уничтоженная встреча — освобождённое рабочее время;
- потерянный шарик — потеря концентрации;
- чашка кофе — возможность продолжить работу.

## 2. Цель проекта

Создать небольшую юмористическую веб-игру для сотрудников IT-компаний.

Игра должна:

- запускаться в браузере;
- быть понятной без обучения;
- визуально напоминать календарь рабочей недели;
- использовать знакомые IT-встречи;
- работать без backend;
- хранить настройки и рекорды локально.

## 3. Целевая платформа

- desktop-first;
- современные браузеры;
- управление клавиатурой и мышью;
- минимальное разрешение: 1366×768;
- рекомендуемое разрешение: 1920×1080;
- полноценная мобильная версия не входит в MVP.

## 4. Основной игровой цикл

1. Игрок запускает уровень.
2. На поле появляется календарь рабочей недели.
3. Календарь заполнен блоками встреч.
4. Шарик находится на платформе.
5. Игрок запускает шарик.
6. Шарик отражается от стен, платформы и встреч.
7. Встречи получают урон и разрушаются.
8. За уничтоженные встречи начисляются очки и освобождённое время.
9. При потере шарика расходуется одна чашка кофе.
10. Если кофе осталось, игрок продолжает.
11. Если кофе закончилось, наступает поражение.
12. После уничтожения всех обязательных встреч уровень завершается победой.

## 7. Игровое поле

### 7.1. Календарная сетка

Поле должно напоминать календарь рабочей недели:

- колонки: Пн, Вт, Ср, Чт, Пт;
- временная шкала: 09:00–18:00;
- линии каждые 30 минут;
- блоки встреч располагаются по дням и времени;
- внешний вид не должен копировать Outlook один в один;
- запрещено использовать логотип Microsoft и фирменные элементы Outlook.

### 7.2. Размер встреч

Высота блока зависит от длительности:

| Длительность | Относительная высота |
|---|---:|
| 15 минут | 1 единица |
| 30 минут | 2 единицы |
| 60 минут | 4 единицы |
| 90 минут | 6 единиц |
| 120 минут | 8 единиц |

Допускается игровое масштабирование, чтобы блоки оставались удобными для столкновений.

### 7.3. Содержимое блока

Внутри блока отображаются:

- название;
- время;
- длительность;
- количество участников;
- необязательная иконка типа встречи.

Пример:

```text
Sprint Planning
10:00–12:00
12 участников
```

Для маленьких блоков показывать только сокращённое название.

## 8. Управление

### Клавиатура

- `A` или `←` — движение влево;
- `D` или `→` — движение вправо;
- `Space` — запуск шарика;
- `Esc` — пауза;
- `R` — перезапуск уровня.

### Мышь

- платформа следует за курсором по горизонтали;
- левый клик запускает шарик.

## 9. Платформа

Платформа находится в нижней части поля.

На платформе отображается:

```text
FOCUS TIME
```

Угол отражения зависит от точки попадания шарика по платформе.

## 10. Шарик

Шарик называется `Task`.

Требования:

- отражается от стен;
- отражается от платформы;
- повреждает встречи;
- постепенно ускоряется;
- имеет ограничение максимальной скорости;
- не должен застревать в строго горизонтальной или вертикальной траектории;
- при зацикливании угол автоматически корректируется;
- после потери появляется на платформе.

## 11. Кофе вместо жизней

Начальное количество:

```text
☕ ☕ ☕
```

Логика:

1. Шарик падает за нижнюю границу.
2. Одна чашка кофе считается выпитой.
3. Количество кофе уменьшается на одну.
4. Игра делает короткую паузу.
5. Новый шарик появляется на платформе.
6. Игрок запускает его снова.
7. Если кофе закончилось, игрок проигрывает.

Пример HUD:

```text
Кофе: ☕ ☕ ☕
```

Случайные фразы при потере шарика:

- «Задача потеряна. Пора за кофе».
- «Без кофе это уже не починить».
- «Ещё одна чашка — и продолжаем».
- «Фокус закончился. Кофе ещё есть».
- «Небольшой перерыв у кофемашины».
- «Продолжаем на кофеине».

Экран поражения:

```text
Кофе закончился

Встречи победили.
```

Кнопки:

- «Заварить заново»;
- «Закончить рабочий день».

## 12. Урон и разрушение встреч

При попадании:

1. Встреча получает урон.
2. Для прочных блоков меняется внешний вид.
3. При нулевом HP блок уничтожается.
4. Игрок получает очки.
5. На экране появляется освобождённое время.

Пример:

```text
+60 минут свободного времени
```

Состояния блока:

1. Обычное.
2. Повреждено.
3. Сильно повреждено.
4. Уничтожено.

Эффекты:

- анимация попадания;
- трещина или снижение яркости;
- частицы при уничтожении;
- всплывающий текст;
- лёгкая тряска экрана для All Hands.

| Встреча | Очки | Освобождённое время |
|---|---:|---:|
| Daily | 100 | 15 минут |
| Team Sync | 200 | 30 минут |
| One-to-One | 250 | 30 минут |
| Refinement | 400 | 60 минут |
| Architecture Review | 500 | 60 минут |
| Sprint Planning | 800 | 120 минут |
| Meeting Without Agenda | 600 | 60 минут |
| All Hands | 1500 | 120 минут |

Множители:

- серия попаданий без касания платформы;
- быстрое уничтожение нескольких встреч;
- прохождение без расходования кофе;
- уничтожение бесполезных встреч;
- сохранённые бонусы.

Пример HUD:

```text
Свободное время: 6 ч 45 мин
Встреч уничтожено: 18
Комбо: x4
Кофе: ☕ ☕
```

## 14. Бонусы

### Async Mode

- создаёт дополнительный шарик;
- символизирует перевод коммуникации в асинхронный формат.

### Decline

- следующее попадание мгновенно уничтожает встречу независимо от HP.

### Espresso Shot

- временно увеличивает ширину платформы;
- снижает скорость шарика;
- действует 10 секунд;
- показывает короткую анимацию чашки эспрессо;
- повторное получение обновляет длительность;
- эффект не складывается;
- после завершения параметры плавно возвращаются к исходным.

Для MVP реализовать все три бонуса.

После MVP можно добавить:

- Could Have Been an Email;
- Delegation.

## 22. Расчёт координат встреч

Координаты должны вычисляться из конфигурации.

Алгоритм:

1. Определить индекс дня.
2. Рассчитать X по ширине колонки.
3. Рассчитать Y по времени начала.
4. Рассчитать высоту по длительности.
5. Ограничить минимальную высоту.
6. Создать статическое физическое тело Phaser.

Функция:

```ts
calculateMeetingRect(
  meeting: MeetingBlockConfig,
  calendarLayout: CalendarLayout
): MeetingRectangle
```

Функция должна быть покрыта unit-тестами.

## 24. localStorage

Сохранять:

- тему;
- открытые уровни;

Не сохранять незавершённую игровую сессию.

## 25. Требования к физике

- шарик не проходит сквозь блоки;
- скорость ограничена;
- угол зависит от точки удара;
- горизонтальные траектории корректируются;
- при застревании угол меняется;
- физика не зависит от FPS;
- после потери шарик возвращается на платформу;
- перед повторным запуском есть короткая пауза;
- Espresso Shot не должен ломать физику.

## 30. Критерии готовности

MVP готов, когда:

1. Работают команды:

```bash
npm install
npm run dev
npm run test
npm run lint
npm run build
```

2. Есть главное меню.
3. Уровень запускается.
4. Поле выглядит как календарь.
5. На поле минимум 20 встреч.
6. Шарик корректно взаимодействует с объектами.
7. Встречи имеют разный HP.
8. Начисляются очки и время.

10. При отсутствии кофе наступает поражение.
